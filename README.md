RLE Advanced – Утилита сжатия текста (RLE) на 7 языках
Мощная утилита сжатия с использованием алгоритма Run‑Length Encoding (RLE) с поддержкой Unicode, потоковой обработки и расширенной статистикой.
Сжимает любые текстовые данные (включая многобайтовые символы) и показывает коэффициент сжатия.

🚀 Возможности
Сжатие – заменяет повторяющиеся символы (от 3 и более) на пару (символ + количество) с экранированием управляющих последовательностей.

Распаковка – полностью восстанавливает оригинальный текст.

Потоковая обработка – не загружает весь файл в память, что позволяет работать с гигантскими файлами.

Поддержка Unicode – корректно обрабатывает UTF-8 (в некоторых реализациях) и многобайтовые символы.

Статистика – вывод коэффициента сжатия и экономии места (в процентах).

Гибкий ввод/вывод – чтение из файла или stdin, запись в файл или stdout.

Цветной вывод – наглядные сообщения об ошибках и успехе.

📖 Использование
Синтаксис (единый для всех языков):

bash
<команда> <режим> <входной_файл> [выходной_файл] [-v]
Режимы
compress – сжать файл.

decompress – распаковать файл.

Опции
-v, --verbose – показать статистику сжатия (коэффициент, экономия).

-h, --help – справка.

Если выходной файл не указан, результат выводится в stdout.
Если входной файл не указан (или указан -), чтение из stdin.

Примеры
bash
# Сжатие файла с выводом статистики
python rle.py compress input.txt output.rle -v

# Распаковка
python rle.py decompress output.rle restored.txt

# Сжатие из stdin в stdout
cat huge.log | python rle.py compress - > compressed.rle

# Только сжатие без статистики
./rle compress data.txt data.rle

🧠 Алгоритм и формат
Сжатие:
Текст сканируется посимвольно. Если серия повторяющихся символов имеет длину ≥ 3, она заменяется на: символ + '\' + десятичное число (например, A\4).
Если серия короче, символы копируются как есть, но символ \ экранируется дублированием (\\).
Это гарантирует однозначную распаковку.

Распаковка:
Сканируется сжатая строка. При встрече \ проверяется, не является ли это экранированным символом \\.
Если после \ идёт цифра – это управляющая последовательность: следующий символ повторяется указанное число раз.

Поддержка бинарных файлов (опционально):
В некоторых реализациях можно переключиться в байтовый режим, но по умолчанию работаем с текстом в кодировке UTF-8.

✨ Дополнительные фичи
Статистика – при включённом флаге -v выводится:

Исходный размер (байт / символы)

Размер после сжатия

Коэффициент сжатия (размер_после / размер_до)

Экономия в процентах

Потоковое чтение – память не ограничена размером файла.

Обработка ошибок – валидация сжатых данных при распаковке.
